# Python 程序设计教学大纲（从零开始 · 面向 AI Agent 开发方向）

## 一、课程基本信息

| 项目 | 内容 |
|------|------|
| 课程名称 | Python 程序设计基础（Agent 开发导向） |
| 适用对象 | 零基础或入门级学习者，有明确 AI/Agent 发展意向 |
| 课程周期 | 4 周（外加 2 天预备周），共约 30 天 |
| 建议投入 | 每日 3–5 小时，其中编码时间不低于 70% |
| 先修要求 | 无。会使用电脑与命令行者可跳过预备周部分内容 |
| 结业产出 | ① 一个不依赖任何框架、独立手写的 mini agent 程序；② 全程 Git 提交记录；③ 一篇源码阅读笔记 |

## 二、课程定位与教学目标

### 课程定位

本课程不以"学完 Python 语法"为目的，而以"获得独立编程能力"为目的。所谓独立，指学习者在课程结束时能够：不依赖教程，从零写出数百行结构清晰、可测试、可维护的程序；能读懂中小型开源项目的主干逻辑；能对 AI 生成的代码作出正确性与质量判断。

### 总体目标

以一个月为期，完成从"零基础"到"具备 Agent 开发所需 Python 工程底座"的跨越，为后续深入 AI Agent 方向（框架原理、上下文工程、工具调用可靠性等）铺平语言层面的道路。

### 分项能力目标

1. **语言能力**：熟练使用 Python 核心数据结构与控制流，理解函数、类、异常、类型注解的设计意图，而非仅记忆语法。
2. **工程能力**：养成虚拟环境、模块化组织、单元测试、断点调试、版本控制五项基本工程习惯。
3. **领域衔接能力**：掌握 HTTP API 调用、JSON 处理、异步并发三项 Agent 开发的刚需技能。
4. **元能力**：建立"先自行调试、再求助工具"的问题解决习惯，学会把 AI 用作教练而非代笔。

## 三、教学理念与方法

### 教学理念

1. **做中学（Project-Driven）**：每个知识点必须在 24 小时内落到代码上，未经手敲的知识视为未学。
2. **自下而上（Bottom-Up）**：先接触具体、可验证的代码行为，再抽象出概念，不提前灌输框架性理论。
3. **刻意练习**：每周设"不查资料手写"环节，暴露真实水平，针对暴露的薄弱点补练。
4. **周关卡制**：每周末设达标检验，未达标者延长该周内容，不赶进度。深度优先于进度。

### AI 使用守则（课程纪律，全程有效）

- 核心代码必须亲手敲入，禁止复制粘贴 AI 生成的整段实现。
- 遇到报错，先独立调试 20 分钟（读报错信息 → 打断点 → 缩小范围），再向 AI 提问。
- 提问时须附上：报错信息、已尝试的排查步骤、自己的猜测。
- AI 的正确用法：让它 review 你的代码并指出坏味道；让它出变式练习题；让它解释你读不懂的源码。
- 每周至少一次，把自己写的代码交给 AI 评审，并将评审意见整理进学习笔记。

## 五、分周教学单元

### 预备周（第 0–2 天）：环境与工具

**教学要点**

- 命令行基础：目录切换、文件操作、运行程序
- 安装 Python（建议 3.12+）与包管理工具（uv 或 pip + venv）
- 编辑器配置（VS Code + Python 插件），学会设置断点
- Git 三板斧：`init`、`add/commit`、查看历史；在 GitHub 建立本课程仓库
- 运行第一个程序，区分"交互式解释器"与"脚本执行"

**达标检验**

- 能在命令行内新建项目目录、创建虚拟环境、安装一个第三方包并在脚本中导入
- 完成第一次 Git 提交并推送到 GitHub

### 第一周：语言地基

**知识点**

1. 变量与基本类型：int / float / str / bool；字符串常用操作与 f-string
2. 控制流：if / for / while；`range`、`enumerate`、`zip`
3. 核心数据结构：list、dict、set、tuple 的增删改查；**各操作的时间复杂度**（本周重点，必须能口述）
4. 推导式：列表推导式与字典推导式
5. 函数基础：参数、返回值、默认值、作用域

**实践作业（每日一题，逐日递进）**

- 词频统计器：读入一段文本，输出出现次数前 10 的词
- 文件批量重命名工具：按规则重命名目录下的所有文件
- 通讯录程序：命令行增删查改，数据存于 dict
- 简易记账本：分类汇总、按金额排序
- 周末综合：任选上述一题，不查任何资料重写一遍

**达标检验**

- 不查资料，30 分钟内独立写出词频统计器
- 能解释：为什么查找用 dict 而不用 list？list 头部插入为什么慢？set 去重的原理是什么？
- 本周每日至少一次 Git 提交

### 第二周：抽象与工程

**知识点**

1. 函数进阶：`*args / **kwargs`、闭包、装饰器（重点：亲手实现计时器与重试器）
2. 面向对象：class、`__init__`、实例方法、`dataclass`；何时用类、何时用函数（设计判断）
3. 异常处理：try / except / finally、自定义异常、"让错误尽早暴露"原则
4. 类型注解：基础标注、`Optional`、`list[dict]` 等容器标注
5. 单元测试：pytest 入门，为已有代码补测试；测试驱动地修一个 bug
6. 调试：断点、单步、观察变量；用调试器而非 print 排查一个真实问题

**实践作业**

- 用装饰器实现：函数执行计时器、失败自动重试器（含最大次数与间隔）
- 将第一周的通讯录程序重构为类，并补全 pytest 测试（覆盖正常与异常路径）
- 定义一套自定义异常，改造记账本的错误处理
- 周末综合：交叉评审——把代码交给 AI 评审，按意见重构一轮，记录前后差异

**达标检验**

- 不看资料手写一个带参数的装饰器
- 通讯录项目测试全部通过，且至少包含 2 个针对异常路径的测试
- 能用断点调试定位一个人为埋入的 bug（可让 AI 出题埋 bug）

### 第三周：连接外部世界

**知识点**

1. 文件与路径：pathlib 读写文本、处理目录树
2. JSON：`json` 模块的序列化与反序列化，嵌套结构的安全取值
3. HTTP 基础：请求与响应、方法、状态码、headers；用 httpx 发起 GET / POST
4. 调用 LLM API：以本地 Ollama 为练习对象，构造请求体、解析流式与非流式响应
5. 异步并发：asyncio 基础——`async / await`、`asyncio.gather`、并发数控制（Semaphore）
6. 健壮性：超时设置、异常重试（复用第二周的重试装饰器）、日志记录（logging 入门）

**实践作业**

- 写一个命令行工具：向本地 Ollama 提问并打印回答（同步版）
- 升级为异步版：并发发送 10 个问题，控制最大并发为 3，统计总耗时对比
- 为上述脚本加入：超时处理、失败重试、logging 日志（替换所有 print）
- 周末综合：批量问答脚本——从 JSON 文件读入问题列表，并发获取回答，结果写回 JSON

**达标检验**

- 不看资料，能说清一次 HTTP 请求从发出到收到响应发生了什么
- 异步批量脚本在人为断网/超时场景下不崩溃，且日志可追溯每个请求的成败
- 能解释：asyncio 的并发和多线程的区别是什么？什么场景适合用它？

### 第四周：综合结业——手写 mini agent

**项目要求**

不使用任何 Agent 框架，仅用 Python 标准库 + httpx，实现一个最小可用的 agent 循环：

1. 接收用户任务（命令行输入）
2. 将任务与工具清单发给模型，由模型决定调用哪个工具
3. 解析模型返回的工具调用（JSON 格式），执行对应的本地函数
4. 将执行结果回填到对话上下文，进入下一轮循环
5. 至少实现 3 个工具：如读文件、写文件、计算器
6. 工程要求：模块化组织（拆为 3 个以上文件）、类型注解、logging 日志、对核心解析逻辑写 pytest 测试、完整 Git 记录

**教学安排**

- 第 1–2 天：设计——画出循环流程图，定义工具注册的数据结构，写出模块划分
- 第 3–5 天：实现——按"先跑通最简循环，再逐个加工具，最后加健壮性"的顺序推进
- 第 6 天：测试与重构——补测试、让 AI 评审、按意见重构
- 第 7 天：源码阅读——选一个小型开源 agent 项目通读主干，对照自己的实现写一篇笔记：它比我多做了什么？为什么？

**达标检验（结业标准）**

- mini agent 能端到端完成一个多步任务（例：读取某文件内容并把统计结果写入新文件）
- 能向他人（或 AI）完整讲解自己每个模块的设计取舍
- 源码阅读笔记能准确描述所读项目的核心循环与自己实现的异同

## 六、考核与评价

| 构成 | 占比 | 说明 |
|------|------|------|
| 过程性考核 | 40% | 每日 Git 提交记录、每周达标检验通过情况 |
| 结业项目 | 40% | mini agent 的功能完整性、代码结构、测试与日志、讲解质量 |
| 源码阅读笔记 | 20% | 是否读懂主干、对比分析是否有自己的判断 |

**评分导向**：结构清晰、命名达意、错误处理完备的 200 行，优于堆砌功能的 800 行。

## 七、教材与参考资料

- 主教材：Python 官方教程（docs.python.org 中文版）——按大纲进度选读对应章节，不通读
- 工具书：官方标准库文档（遇到即查，培养查文档习惯）
- 第三方库文档：httpx、pytest 官方文档
- 明确不推荐：在本阶段购买视频大课、通读《流畅的Python》等进阶书（第二个月之后再说）

## 八、给学习者的话

这份大纲最大的敌人不是难度，而是两个诱惑：一是"看懂了就往下走"（看懂不等于会写，一切以手敲为准）；二是"让 AI 替我写完"（那样一个月后你收获的只是一个别人的项目）。第四周的结业项目在第一周看会觉得遥不可及，这是所有从零开始的人的共同感受——按周推进，逐关通过，它会自然长出来。祝顺利。
